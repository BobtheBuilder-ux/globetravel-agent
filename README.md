Globe Elite Travel Agency — Next.js

Overview
- Migrated from Vite + React to Next.js (App Router) with TypeScript and Tailwind CSS.
- Preserves original `src/components` structure and styling while adopting Next.js features.

Getting Started
- Install dependencies: `npm install`
- Development server: `npm run dev` then open `http://localhost:3000` (or port announced in terminal)
- Lint: `npm run lint`
- Typecheck: `npm run typecheck`
- Production build: `npm run build` then `npm run start`

Project Structure
- `src/app/` — Next.js App Router
  - `layout.tsx` — global layout and metadata
  - `page.tsx` — homepage rendering existing components
  - `globals.css` — global styles with Tailwind v4 (`@import "tailwindcss"`)
- `src/components/` — existing feature components (Navigation, Hero, Packages, Services, WhyChooseUs, Contact, Footer)
- `public/` — static assets (if added)
- `.next/` — build output (ignored)

Configuration
- `next.config.js`
  - Enables strict mode and remote image optimization for `images.pexels.com`
- `tsconfig.json`
  - Sets Next.js-compatible compiler options and path alias `@/* -> ./src/*`
- `postcss.config.js`
  - Uses Tailwind v4 via `@tailwindcss/postcss`
- `eslint.config.js`
  - Flat ESLint config (ignores `.next` and `node_modules`)

Environment Variables
- Converted from Vite `import.meta.env` to Next.js `process.env`
- Public values are prefixed with `NEXT_PUBLIC_`
  - `NEXT_PUBLIC_SUPABASE_URL`
  - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- Update code to read with `process.env.NEXT_PUBLIC_*` when integrating Supabase or other services

Routing, SSG & SSR
- App Router (`src/app`) is used
- The homepage (`/`) is statically generated by default
- Add server components, data fetching, or route handlers under `src/app/*` as needed

Image Optimization
- Replaced `<img>` with Next.js `Image` where applicable
- Remote patterns for Pexels configured in `next.config.js`

Scripts
- `dev` — starts Next dev server
- `build` — creates production build
- `start` — runs production server
- `lint` — runs ESLint
- `typecheck` — runs TypeScript without emit

Notes
- Removed Vite-specific files: `index.html`, `vite.config.ts`, `src/main.tsx`, `src/App.tsx`, `vite-env.d.ts`, Tailwind v3 config
- Tailwind v4 is used; a `tailwind.config.js` file is not required unless customizing theme
